# Flexible Itinerary · 灵活行程

[English version](./README.md)

页面地址：https://yixiaoer.github.io/flexible-itinerary/

一个为自由旅行准备的灵活行程规划工具。你可以先粗略排出每天想去哪里，再逐步细化到上午、下午、晚上，甚至精确到分钟；也可以锁定演出、交通、日出等关键安排，再围绕它们调整其它活动。

## 核心能力

- **时间颗粒可粗可细**：同一个行程里，可以只把活动放在某一天，也可以安排到“上午 / 下午 / 晚上”，还可以精确到具体开始和结束时间。可以自由规划，也可以逐步精确。
- **灵活日期与时段**：可以只规划“几天”，也可以填写具体开始日期；不确定日期时也能先把行程搭起来。
- **关键安排可以锁定**：演唱会、飞机、火车、日出、预约餐厅、展览门票等不能随便变动的活动，可以锁定当前安排，避免后续拖拽或调整时误改。
- **拖拽式行程看板**：地点和活动可以在候选区与每天的行程之间拖拽，也可以跨天移动、当天排序、拖回候选区，让排程像整理卡片一样直观。
- **地图视图**：已解析的地点会显示在地图上，也可以直接搜索地点并加入候选活动，方便从空间分布上检查路线是否合理。
- **本地多行程管理**：支持多个行程档，本地自动保存；新建行程会立即入库，点击行程卡片即可切换，也可以拖动调整行程顺序。
- **版本与备份**：当前行程会自动保存，也可以手动“保存版本”作为快照；支持单个行程 JSON 导入和导出。
- **行程概览与状态标签**：支持已出行、将出行、出行中、长期计划等状态；Review / 行程评估可以查看天数、活动数量、候选活动、总时长和风险提示，更完整的 AI 评估能力待开发。
- **AI 辅助规划（待开发）**：未来希望支持 AI 生成行程草稿、补充候选地点、根据天气或突发情况重排，并辅助检查营业时间、路线冲突等。

## 怎么用

使用者不需要安装或配置本地环境。打开部署后的 GitHub Pages 页面即可使用。

1. **新建或打开行程**：点击顶部“新建行程”创建新档；打开“我的行程”后，也可以直接点击已有行程卡片切换。
2. **填写基础信息**：在左侧填写目的地、天数、可选日期、想去的地方和偏好。修改后会自动更新当前行程。
3. **管理候选活动**：想去的地方会进入候选区，并自动尝试解析地图位置；解析过的地点会随行程保存。
4. **拖拽排程**：把候选活动拖到某一天，也可以在天内排序、跨日移动，或拖回候选区；每一天还可以选择“清空当天”，把当天活动移回候选但保留这一天。
5. **调整时段**：活动支持上午、下午、晚上、灵活。拖拽位置会自动影响时段，手动设置晚上后会形成新的边界。
6. **编辑活动**：点击活动可编辑标题、地点、时长、时间、备注、锁定状态等。时长可以留空。
7. **查看地图和评估**：顶部切换到“地图”查看地点分布；切换到 Review / 行程评估查看行程概览和风险提示。更完整的 AI 评估能力待开发。
8. **保存版本或备份**：当前行程会自动保存；需要保留快照时，可以点击“保存版本”。也可以对单个行程导出 JSON，或导入 JSON 恢复。
9. **清空或重新开始**：顶部“清空安排”会保留目的地并清空其它安排；“新建行程”会创建一个新的空白行程档。

## 功能状态与 Roadmap

### 时间颗粒与日期

- [x] 活动可以只放在某一天，不强制填写具体时间。
- [x] 活动可以设置为上午、下午、晚上或灵活。
- [x] 活动可以精确到具体开始和结束时间。
- [x] 活动时长可以留空；内部计算会使用临时默认值，但不会把默认值强行写回活动。
- [x] 出行日期可以模糊：可以只填天数，也可以同时填开始日期。
- [x] 拖拽后会根据位置推断上午、下午或晚上，并保持时段边界一致。

### 锁定与活动编辑

- [x] 支持锁定活动，用于演唱会、飞机、火车、日出、预约餐厅等不可随意变动的安排。
- [x] 锁定后，移动和自动排程逻辑会保留关键字段。
- [x] 活动编辑器支持标题、地点、备注、时段、时间、锁定、可选状态等字段。
- [x] 支持灵活活动；灵活活动不受自动时段边界影响。

### 拖拽式行程看板

- [x] 候选活动可以拖到任意一天。
- [x] 已安排活动可以在同一天内拖拽排序。
- [x] 已安排活动可以跨天移动。
- [x] 已安排活动可以拖回候选区。
- [x] 可以清空某一天，把当天活动移回候选区，同时保留这一天。
- [x] 可以删除某一天，并把当天活动移回候选区。
- [x] 未锁定活动支持整张卡片拖拽。

### 地图视图与地点解析

- [x] 想去的地方会自动生成候选活动。
- [x] 可解析的地点会自动补充经纬度。
- [x] 已解析地点会尽量从行程数据中恢复，刷新或修改其它输入时不会重复解析。
- [x] 地图视图可以显示已解析地点。
- [x] 地图视图支持直接搜索地点并加入候选活动。
- [x] 地点名与地图坐标解耦：可以自由修改活动标题（比如改成「英文名 + 中文名」），不会把同一个地点再次加进候选；手动添加的活动也会自动按标题尝试解析坐标。
- [x] 活动编辑器里新增「在地图上查找」：可以为已有活动补上地图坐标，或重新选择地点。
- [ ] 地图 marker 拖动后反向更新地点。

### 本地多行程管理、版本与备份

- [x] 本地优先的行程编辑器，核心数据保存在浏览器 `localStorage`。
- [x] 支持多行程库：可查看、点击切换、复制、删除、导入 JSON、导出 JSON。
- [x] 新建行程会立即创建对应行程档，切换到其它行程后不会丢失。
- [x] 当前打开的行程会自动保存，并同步回“我的行程”里的对应卡片。
- [x] “我的行程”支持用右侧拖拽上下调整顺序。
- [x] 可以清空当前安排：保留目的地，清空日期、想去的地方、偏好、候选和每天安排。
- [x] `保存版本` 会额外创建一个独立快照，不覆盖当前正在编辑的行程。
- [ ] 自定义版本历史对比、回滚和命名版本尚未实现；当前只有“保存版本”快照。
- [ ] 账号、密码、Apple ID、iCloud 或跨设备同步。
- [ ] 多人协作。
- [x] 手机 Web 页面基础适配：行程输入会折叠成抽屉，看板、候选区、编辑弹窗等都可在手机上使用。

### 行程概览与状态标签

- [x] 行程状态标签支持已出行、将出行、出行中、长期计划。
- [x] 行程状态支持按日期自动推断，也可以手动修改。
- [x] Review / 行程评估可以展示天数、活动数量、候选活动、总时长和风险提示等概览。

### AI 相关

- [ ] AI 配置尚未开放；设置里的接口地址、API Key、模型、随机度、额外系统提示目前是禁用占位。
- [ ] AI 生成完整行程尚未正式启用。
- [ ] AI 单日重排 / 全局重排尚未正式启用。
- [ ] AI 建议更多候选活动尚未正式启用。
- [ ] LLM 营业时间检查和复杂证据层尚未正式启用。
- [ ] 流式 AI 输出尚未实现。

## 隐私

当前行程、行程库、备注和设置都保存在你的浏览器 `localStorage`。打开 DevTools -> Application -> Local Storage 可以查看或清除。
